# Receipt Bridge

Android print service for generic ESC/POS thermal receipt printers. Receipt
Bridge appears in Chrome and other Android print dialogs and sends print jobs
to configured printers over Ethernet, Bluetooth Classic, or USB/OTG.

Developed by **Aung Kyaw Thu**.

## Features

- Android system `PrintService` integration
- Print directly from Chrome and other Android applications
- Ethernet/Wi-Fi printing through raw TCP, normally port `9100`
- Bluetooth Classic printing through ESC/POS SPP/RFCOMM
- USB host/OTG bulk printing
- 72 mm, 80 mm, and 100 mm roll-paper options
- 203 DPI and 300 DPI printer profiles
- Adjustable font and content scale
- Independent side-margin control
- Compact paper-saving mode that removes unused white space
- Configurable print density
- Automatic paper feed and cutter controls
- Cash-drawer connector 1 and 2 pulses
- Optional printer alarm before or after printing
- Multiple-copy support through the Android print dialog
- Persistent printer connections
- Direct test-page printing
- Local failed-job cache with a 50 MB limit
- PrintService diagnostics
- In-app privacy policy

## Supported Printers

Receipt Bridge targets generic printers that understand ESC/POS raster and
control commands. Hardware-specific features such as the cutter, buzzer, and
cash drawer depend on printer support.

Typical standard 80 mm printers use a 72 mm, 576-dot print head at 203 DPI.
Select 300 DPI only when the printer hardware is genuinely a 300 DPI model.

## Getting Started

### Requirements

- Flutter SDK
- Android SDK and Android Studio JDK
- An Android phone or tablet with system printing support
- A compatible ESC/POS thermal printer

### Install dependencies

```powershell
flutter pub get
```

### Run on an Android device

Enable USB debugging, connect the device, and run:

```powershell
flutter run
```

### Build an APK

```powershell
flutter build apk
```

The APK is generated at:

```text
build/app/outputs/flutter-apk/app-release.apk
```

The current Android release configuration uses a debug signing key. Configure
proper release signing before publishing the app.

## Configure a Printer

1. Open Receipt Bridge.
2. Tap **Add printer**.
3. Select Ethernet, Bluetooth Classic, or USB/OTG.
4. Enter or select the printer address.
5. Choose the paper and output settings.
6. Save the printer.
7. Press **Print test page** to verify the connection.

### Ethernet

Enter the printer's current IP address and raw printing port. Most ESC/POS
network printers use port `9100`. The phone and printer must be reachable on
the same network.

### Bluetooth

Pair the printer in Android Bluetooth settings first. Receipt Bridge attempts
insecure SPP, secure SPP, advertised UUIDs, and RFCOMM channel 1 for broad
compatibility with generic receipt printers.

### USB

Connect the printer through a USB OTG adapter and approve Android's USB access
prompt. The printer must expose a compatible bulk output endpoint.

## Enable Android Printing

1. Open Receipt Bridge.
2. Tap **Enable in Android settings**.
3. Select **Receipt Bridge**.
4. Enable **Use print service**.

The menu path on many devices is:

```text
Settings > Connected devices or More connections > Printing > Receipt Bridge
```

## Print from Chrome

1. Open the page or voucher in Chrome.
2. Select **Print**.
3. Choose the configured Receipt Bridge printer.
4. Select the required 72 mm, 80 mm, or 100 mm roll size.
5. Choose the copy count.
6. Press the print button.

The Chrome preview is generated by the source webpage before Receipt Bridge
receives the document. Website preview margins must be corrected in the
website's `@media print` CSS. Receipt Bridge can crop and scale the physical
output, but it cannot modify Chrome's preview.

## Recommended 80 mm Settings

| Setting | Recommended value |
| --- | --- |
| Roll paper | 80 mm |
| Receipt layout | Full paper |
| Font and content size | 85% |
| Side margin | 1 mm |
| Resolution | 203 DPI |
| Paper saving | Compact |
| Density | 2 or 3 |
| Keep connection alive | On |

## Architecture

- **Flutter UI:** printer configuration, testing, cache controls, diagnostics,
  and privacy information
- **Android PrintService:** printer discovery and Android Print Spooler job
  handling
- **PDF renderer:** converts Android print documents into monochrome raster
  data
- **ESC/POS transport:** sends raster and hardware-control commands through
  TCP, Bluetooth RFCOMM, or USB bulk transfer

All Android `PrintJob` state operations run on the main application thread.
PDF rendering and printer I/O run on worker threads.

## Privacy

Receipt Bridge does not include advertising or analytics and does not upload
printer settings or print documents to a developer-operated server.

- Printer configuration is stored in Android private app storage.
- Successful print documents are deleted from the app cache.
- Failed documents may remain locally within the 50 MB cache limit.
- Users can clear the cache from the home screen.
- Print data is sent only to the printer destination selected by the user.

The complete Privacy Policy is available inside the app from the shield icon
or the footer link.

## Troubleshooting

### Print test succeeds, but Chrome printing fails

- Confirm Receipt Bridge is enabled in Android printing settings.
- Cancel old failed jobs instead of restarting them.
- Create a new Chrome print job and select the printer again.
- Check the in-app **Android PrintService log**.

### Ethernet connection fails

- Confirm the printer IP address has not changed.
- Confirm the port, normally `9100`.
- Ensure the phone and printer are on the same LAN.
- Avoid guest Wi-Fi networks that block device-to-device traffic.

### Bluetooth connection fails

- Restart the printer.
- Forget and pair the printer again in Android settings.
- Common pairing PINs are `0000` and `1234`.
- Allow the Nearby devices permission.

### Output is clipped or shifted

- Use 203 DPI for a standard 576-dot printer.
- Select **Full paper** layout.
- Adjust font/content size and side margin independently.
- Create a new Chrome job after changing printer settings.
